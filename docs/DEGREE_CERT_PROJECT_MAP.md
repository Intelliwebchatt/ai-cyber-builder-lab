# Degree, certification, and project map

The degree provides structured knowledge. Certifications provide recognizable external signals. Projects provide inspectable evidence. The three should reinforce one another.

| Course or knowledge area | Independent portfolio evidence | Certification connection |
|---|---|---|
| Introduction to scripting | Python log parser, IOC normalizer, tests, and error handling | Security automation foundation |
| Application development | Small secure application with authentication and authorization tests | Application-security foundation |
| Fundamentals of IT | System inventory, architecture diagram, asset classification | Entry-level IT and security concepts |
| Operating systems | Windows and Linux hardening lab with before-and-after validation | Security+ foundation |
| Computer networks | Segmented network diagram, packet analysis, DNS and firewall exercise | Cisco networking or Security+ foundation |
| Databases | Security-events schema, safe queries, access design, audit logging | Data protection and analyst skills |
| Cybersecurity foundations | NIST-based risk assessment and security plan | Security+ foundation |
| Network security | Firewall policy, segmentation, network telemetry, and validation | Security+ and analyst preparation |
| Operating-system security | Hardening scripts, least privilege, patch and configuration evidence | Security+ and blue-team preparation |
| Application security | OWASP assessment, remediation pull request, automated security tests | AppSec preparation |
| Cyber defense | Mini SOC, alert triage, detection rule, and ATT&CK coverage | BTL1 or analyst preparation |
| Incident response | Case file, timeline, evidence log, containment, recovery, lessons learned | BTL1 or analyst preparation |
| Security assessment and auditing | Control matrix, evidence requests, findings, and management response | GRC and audit preparation |
| Risk management | Risk register, business-impact analysis, treatment plan | GRC preparation |
| Enterprise security | End-to-end architecture and security review | Analyst and engineering preparation |
| AI literacy | AI system inventory and tool-evaluation record | AI-security foundation |
| Responsible AI | AI acceptable-use policy and risk case study | AIGP foundation |
| AI in practice | Reproducible AI workflow with evaluation and human-review controls | AI-security and automation evidence |
| Data policy and decision making | Data classification, retention, privacy, and governance plan | AIGP and GRC foundation |

## Publication rule

Do not publish graded coursework. Build a separate synthetic extension that demonstrates the same skill and can be explained without course-owned material.
