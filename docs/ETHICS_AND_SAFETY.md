# Ethics, safety, privacy, and academic integrity

## Authorized use only

This repository is for defensive education and portfolio development. Every lab must be owned by the repository owner, intentionally isolated for testing, or covered by explicit permission.

## Prohibited content

Do not commit:

- real criminal-investigation evidence or active-case information
- victim, witness, suspect, inmate, customer, or patient information
- authentication credentials, API keys, session tokens, or private keys
- employer or client data without written authorization
- exploit instructions aimed at real targets
- deployable malware, destructive payloads, credential theft, or stealth mechanisms
- verbatim graded assignments, answer keys, restricted lab files, or instructor-owned solutions

## Synthetic scenarios

Fictional organizations, names, timestamps, domains, and network indicators must be clearly labeled. Use IANA-reserved example domains and documentation IP ranges whenever possible.

## Evidence integrity

For investigation projects:

- identify the evidence source and collection time
- preserve an original copy when appropriate
- calculate a cryptographic hash
- record every transformation
- distinguish observed facts from inference
- document uncertainty and alternate explanations

## AI-assisted work

AI output is not evidence. It may propose hypotheses, code, queries, or report language, but the repository owner must validate each result against the underlying data. AI assistance must be disclosed in each completed project.

## Academic integrity

Follow the rules of the applicable course and instructor. Publish only original extensions created for the portfolio. When a course artifact cannot be published, create a new synthetic scenario demonstrating the same skill.
