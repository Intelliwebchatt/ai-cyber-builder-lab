# Career evidence map

This map translates prior professional strengths into cybersecurity proof that an employer can inspect. Historical licenses and training are context, not current credentials.

| Target role | Transferable foundation | Repository proof still required |
|---|---|---|
| Cybercrime or investigative analyst | Major-case investigation, interviews, evidence handling, warrants, interagency coordination, report writing | OSINT workflow, digital-evidence case, SQL or data analysis, intelligence brief |
| CJIS or public-safety security analyst | Sensitive records, access responsibilities, public-safety operations, policy enforcement, incident command | Access-control assessment, audit log review, incident plan, CJIS policy crosswalk |
| Digital-forensics analyst | Crime-scene processing, chain of custody, evidence reasoning, courtroom communication | Disk and memory artifacts, forensic-tool workflow, hashes, timeline, defensible report |
| SOC or incident-response analyst | Incident leadership, triage, documentation, decision-making under pressure | Networking, Windows and Linux logs, SIEM investigation, containment, recovery validation |
| Insider-threat analyst | Internal investigations, interviewing, behavioral indicators, supervisory judgment | IAM analysis, user activity timeline, privacy-aware investigation plan, risk briefing |
| Threat-intelligence or OSINT analyst | Informant development, multi-agency intelligence, fraud and identity-theft investigations | Source evaluation, IOC enrichment, ATT&CK mapping, analytic confidence statement |
| GRC or technology-risk analyst | Safety audits, policy revision, budget ownership, regulatory awareness, staff training | NIST control assessment, risk register, evidence requests, findings and remediation plan |
| OT or critical-infrastructure security analyst | Electrical systems, generators, batteries, panels, fiber, diagrams, testing, safety procedures | OT network model, segmentation design, asset inventory, monitoring plan, safe recovery exercise |
| Application-security analyst | Full-stack development, APIs, serverless functions, databases, cloud services | Threat model, secure-code review, automated tests, vulnerability remediation, validation |
| AI security or governance analyst | AI application development and planned generative-AI concentration | AI inventory, NIST AI RMF assessment, OWASP GenAI testing, data policy, evaluation evidence |

## Readiness rule

A role moves from an interest to a credible target when the repository contains at least two substantial, independently explainable projects matching its daily work.
